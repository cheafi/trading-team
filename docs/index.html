� Honest Assessment The quality model uses 3 features (hour, weekday, side) — a session-direction prior, not deep trade intelligence. The regime model is trained but NOT used in live decisions. Regime detection is rule-based. Training via the API relearns from existing backtests — it does NOT download fresh data.

🚀 Quick Start

1. Clone & Configure

git clone https://github.com/cheafi/Trading-bot-CC.git
cd Trading-bot-CC
cp .env.example .env          # fill in your keys (optional — dry_run works without)
chmod +x scripts/*.sh

2. Start Full Stack

# Start all 4 Docker services
./scripts/start.sh
# — or —
docker compose up -d

3. Open Dashboard

open http://localhost:3000     # Next.js control panel

4. Backtest All Strategies

./scripts/backtest-all.sh 20240101-20260101

# Or single strategy
docker compose run --rm freqtrade backtesting \
  --config /freqtrade/config/config_backtest.json \
  --strategy AdaptiveMLStrategy \
  --strategy-path /freqtrade/user_data/strategies \
  --timerange 20240101-20260101 --timeframe 5m \
  --enable-protections --export trades

5. Retrain ML Models

# Via API (triggers background job with real-time logs)
curl -X POST http://localhost:3001/api/ml/train

# Or directly inside the Freqtrade container
docker compose exec freqtrade python \
  /freqtrade/user_data/strategies/ml_optimizer.py --retrain

6. Check Freqtrade API

# Health check
curl http://localhost:8080/api/v1/ping

# Profit summary (creds: freqtrader / SuperSecure123)
curl -u freqtrader:SuperSecure123 http://localhost:8080/api/v1/profit

# Open trades
curl -u freqtrader:SuperSecure123 http://localhost:8080/api/v1/status

7. Tail All Logs

docker compose logs -f                  # all services
docker compose logs -f freqtrade         # trading engine only
docker compose logs -f agent-runner      # agents only

8. Stop Everything

./scripts/stop.sh
# — or —
docker compose down

🤖 Discord Bot Setup

Optional but recommended — get real-time alerts, risk notifications, and trade signals in Discord.

Create Discord Application

Go to Discord Developer Portal → "New Application" → name it "CC Trading Bot"

Create Bot

Bot tab → Add Bot → Copy TOKEN. Enable MESSAGE CONTENT INTENT under Privileged Gateway Intents.

Invite Bot to Server

OAuth2 → URL Generator → Scopes: bot, applications.commands → Permissions: Send Messages, Embed Links, Read Message History → Copy URL → Open in browser → Select server → Authorize.

Configure .env

DISCORD_BOT_TOKEN=your_bot_token_here
DISCORD_CHANNEL_NAME=trading-cc

Test Connection

Restart: docker compose restart agent-runner
Check: curl http://localhost:3001/api/discord/status
In Discord: !help → Bot responds with commands

📊 Backtesting

# Backtest all strategies for a date range
./scripts/backtest-all.sh 20240101-20260101

# Backtest specific strategy
docker compose run --rm freqtrade backtesting \
  --config /freqtrade/config/config_backtest.json \
  --strategy A52Strategy \
  --strategy-path /freqtrade/user_data/strategies \
  --timerange 20240101-20260101 \
  --timeframe 5m \
  --enable-protections --export trades

# Download historical data first (if needed)
docker compose run --rm freqtrade download-data \
  --config /freqtrade/config/config.json \
  --pairs ETH/USDT:USDT BTC/USDT:USDT SOL/USDT:USDT BNB/USDT:USDT XRP/USDT:USDT DOGE/USDT:USDT \
  --timeframes 5m 15m 1h \
  --timerange 20240101-20260101

# Trigger backtest via API
curl -X POST http://localhost:3001/api/backtest/run \
  -H "Content-Type: application/json" \
  -d '{"strategy": "all", "timerange": "20240101-20260101"}'

# Get backtest results
curl http://localhost:3001/api/backtest/results
